Quickstart: Compressa Platform

This guide shows how to deploy Compressa Platform for model inference.

Requirements​

Deploying Compressa Platform requires a server with 2 GPUs.
The requirements for GPU versions and server setup can be found on this page.

Setup​

First, clone the repository with the configuration:

git clone -b gm/compressa-platform git@github.com:compressa-ai/compressa-deploy.git
cd compressa-deploy

The repository contains two main files that we’ll configure:

  • .env
  • docker-compose.yml

Set up the IDs for GPUs in the .env file:

DOCKER_GPU_IDS_1=<ID1>
DOCKER_GPU_IDS_2=<ID2>

With the default configuration, the services will be deploed at 8080.

If you need to modify these, update the port mappings in docker-compose.yml for container compressa-api-nginx accordingly.

Setup storage​

By default, the containers use the following storage path:

  • compressa - ./data/compressa
    This directory should have 777 permissions, which can be set via: 
    chmod 777 -R ./data/compressa

You can change the storage path in docker-compose.yml.

Then, you can run the solution with:

docker compose up --build

Deploy Inference and Embedding Models​

When the services are running, we need to deploy LLM models to Compressa. The solution uses the LLama3-8B model for chat and the SFR-Embedding-Mistral model for embeddings.
Models can be deployed using the REST API or using Swagger's UI.

The REST APIs are available at:

  • SERVER_NAME:8080/pod-1/api/
  • SERVER_NAME:8080/pod-2/api/

Swagger’s UI is available at:

  • SERVER_NAME:8080/pod-1/api/docs
  • SERVER_NAME:8080/pod-2/api/docs

Here are the commands to deploy models using curl:

Add Compressa-Qwen2.5-14B model in Compressa:​

curl -X 'POST' \
  'http://localhost:8080/pod-1/api/v1/models/add/?model_id=compressa-ain%2FQwen2.5-14B' \
  -H 'accept: application/json' \
  -d ''

Add embedding model in Compressa:​

curl -X 'POST' \
  'http://localhost:8080/pod-2/api/v1/models/add/?model_id=Salesforce%2FSFR-Embedding-Mistral' \
  -H 'accept: application/json' \
  -d ''

When downloading is finished, we can deploy the models:

Deploy Compressa-LLM​

curl -X 'POST' "$SERVER_NAME/pod-1/api/v1/deploy/" \
  -H 'accept: application/json' \
  -H 'Authorization: Bearer EMPTY' \
  -H 'Content-Type: application/json' -d '{
  "model_id": "compressa-ai/Qwen2.5-14B",
  "served_model_name": "Compressa-LLM",
  "dtype": "float16"
}'

Deploy embedding model​

curl -X 'POST' "$SERVER_NAME/pod-2/api/v1/deploy/" \
  -H 'accept: application/json' \
  -H 'Authorization: Bearer EMPTY' \
  -H 'Content-Type: application/json' -d '{
  "model_id": "Salesforce/SFR-Embedding-Mistral",
  "served_model_name": "Compressa-Embedding",
  "dtype": "float16"
}'

When the models are deployed, the server is ready to use!
